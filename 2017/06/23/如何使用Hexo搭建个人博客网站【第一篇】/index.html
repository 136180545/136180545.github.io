写在前面

相信很多同学都曾想过搭建一个属于自己的博客网站，闲暇时写一写博客，我很早就想搭建一个自己的博客网站了，起初是想自己写代码开发一个，后来发现成本实在太高，不仅需要时间，还要过硬的开发技术。直到有一天，我遇到了Hexo这个开源的博客框架，如获至宝，于是便迅速着手开始搭建。可能是我把这件事想的太简单了，过程中遇到了无数个坑，折腾了许久才最终完成。

不得不说，我自己还稍有一点开发基础，但在使用Hexo搭建博客的过程中，有很多新概念也是第一次接触到，有很多问题也是摸索了很久才找到解决方法。作为一款开源框架，Hexo是非常优秀的，但对他的使用者来说却并不友好，我大概遍历了一下网上能够找到的教程，发现即使很多有软件开发功底的同学也是碰到了不少坑，各种奇奇怪怪的坑，把大家折腾的怪难受的。这只能说Hexo的产品化程度还不够高，用户体验有待完善。

适用人群

在我将自己的博客搭好之后，决定写下教程，希望能让后来有兴趣搭建博客的同学少走一些弯路。本篇教程分为三篇，分阶段去达成我们的目标。
我会将教程写的非常详细，主要面向0基础的同学，甚至连Hexo是什么都不了解的同学也能通过本教程的学习最终能成功搭建自己的个人博客。有基础的同学，或者希望快速搭建的同学，可以去搜索其他简明的教程或者查看官方文档Hexo官方文档

教程宗旨

本教程是我自己在搭建过程中遇到问题，解决问题的过程。作为第一版本的教程，他是不完善的，也无法面面俱到，对于这些不完善的地方，我之后会找到很多小白同学，尤其是没有任何软件基础知识的同学，让他们按照本教程一步步去实践，我会在过程中不断询问他们的意见，接收用户的反馈，不断优化完善，提升阅读的体验，最终迭代升级为一个具有普适意义的教程，让更多非专业的同学能够借助此教程搭建自己的博客网站，一起学习知识、分享经验。

名词解释

对于没有软件基础知识的同学，首先要明确一些基本概念，就是接下来用到的这些工具是什么。你也可以先不急着看这个部分，等到后面的安装中有不理解的时候，再回过头来查阅。
1.Hexo
一款非常好用的博客框架，运行速度快，开源免费。

可能有的同学不太了解框架的概念，这里可将Hexo简单理解为一个打包好的软件，像安装QQ一样按照他的特定步骤安装好以后可以免费使用

2.Github
一个面向开源及私有软件项目的托管平台，也是全网最大男性同性交友社区。

这里可简单理解为一台云服务器，我们可以把自己博客网站的代码部署在上面，二十四小时不停地运行，让别人可以访问你的博客

3.Git
Git是一款免费、开源的分布式版本控制系统。

这里可简单理解为搬运工，帮我们从Github云服务器上把Hexo搬到自己的电脑上，等你把Hexo在自己的电脑上配置好以后，再帮你搬到Github

4.Node.js
Node.js是JavaScript代码的运行环境，基于chrome V8引擎。

这里可将整个网站代码整体理解成一列火车，火车只能在铁轨上运行，Node.js就相当于那一条铁轨

在电脑本地搭建网站

首先要将博客网站在你自己的电脑上安装好，并调试好，成功以后再将本地安装好的网站部署到云服务器上，也就是github上。博客网站就是一个小软件，任何人想要查看的时候都可以通过浏览器的域名从云端的服务器上载入这个程序，这样就可以很方便的从电脑和手机上看到你的网站了。

这里我以 Windows 64位 的系统为例讲解如何在电脑本地搭建（注：下面所提供的软件下载地址都是64位的软件）。

将本地搭好的网站部署到Github

本地搭建好博客，只成功了一半，若想要更多的人能够访问你的博客，则需要将在本地搭建好的博客文件部署到云服务器上，也就是github上。

配置Github

Github是一台二十四小时运转的云服务器，我们要先注册账号，并新建我们博客代码文件的仓库。github给每一位用户的免费使用空间是300M，300M的空间足够存放博客的代码文件了。

1.首先打开https://github.com/网站，注册github，github虽然是纯英文网站，但仅仅只是语言不同，登录注册的流程是使用的方法与其他网站没有区别，按照步骤使用邮箱注册并激活即可。

2.创建一个代码仓库，如下图，点击 + 再选择 New repository

3.来到创建仓库的页面，如下图，这里有一个最重要的点就是，仓库名必须是你的：用户名.github.io，就是之前注册时的Username，写法按照这样 username.github.io ，要是不记得你的用户名可以点击你的头像，找到下拉列表里的 Signed in as 后面的就是你的用户名，然后写上一点仓库的描述，如“Blog仓库”，勾选READEM，再点击创建：

4.创建成功会跳转到仓库的页面，点击 Settings 会跳转到设置页面：
